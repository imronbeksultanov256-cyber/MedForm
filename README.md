# MEDCHEF UNIFORM — интернет-магазин формы

Премиальный mobile-first магазин медицинской и поварской формы с оформлением заказа через WhatsApp.

## Стек

Next.js 14 (App Router) · React 18 · TypeScript · Tailwind CSS · Framer Motion · Lucide Icons

## Запуск локально

```bash
npm install
npm run dev
```

Откройте http://localhost:3000

Сборка для продакшена:

```bash
npm run build
npm start
```

## Структура

- `app/page.tsx` — главная страница (Hero-слайдер, преимущества, категории, каталог, о нас, FAQ, контакты, футер)
- `app/product/[slug]/page.tsx` — страница товара с галереей, выбором размера и кнопкой заказа в WhatsApp
- `app/admin/page.tsx` — админ-панель (доступна по `/admin`)
- `components/` — все UI-блоки
- `lib/products.ts` — исходный список товаров (сид)
- `lib/store.ts` — хранилище товаров поверх `localStorage`, используется админ-панелью
- `lib/whatsapp.ts` — сборка ссылки и текста сообщения для WhatsApp
- `public/products/` — фотографии товаров

## Номер WhatsApp

Указан в `lib/whatsapp.ts` (`WHATSAPP_PHONE = "996559949995"`). Измените константу, если номер поменяется.

## Админ-панель

Доступна по адресу `/admin` (ссылка также есть в футере сайта). Позволяет без кода:

- добавлять и удалять товары;
- менять фото (путь к файлу в `public/products/` или внешнюю ссылку), цену, описание, материал, размеры;
- **менять и добавлять цвета** товара (формат `Название:#HEX`, через запятую) — по умолчанию у каждого товара уже есть 4 цвета: Белый, Тёмно-синий, Хаки, Чёрный;
- скрывать/показывать товар в каталоге;
- отмечать «Новинка» и «Хит продаж».

## Оформление заказа

В каталоге у карточки товара теперь одна кнопка — **«Купить»**, которая ведёт на страницу товара. Там покупатель читает описание, выбирает цвет, размер и количество, после чего кнопка «Заказать через WhatsApp» становится активной и открывает WhatsApp с готовым сообщением:

```
Здравствуйте! 👋
Хочу оформить заказ на форму.

📋 Информация о товаре:
• Модель: ...
• Категория: ...
• Пол: ...
• Размер: ...
• Цвет: ...
• Количество: ...
• Цена: ... сом
```

Общая кнопка «Написать» (шапка, футер, плавающая кнопка) отправляет короткое приветствие «Здравствуйте!» без деталей заказа.

**Важно:** в текущей версии данные админ-панели хранятся в `localStorage` браузера — это готовое MVP-решение без бэкенда, которое сразу работает и удобно для тестирования и одного администратора. Изменения видны только в том браузере, где они сделаны. Для полноценной работы нескольких сотрудников и synchронизации между устройствами рекомендуется подключить базу данных (например, Supabase/Postgres) и заменить функции в `lib/store.ts` на запросы к API — верстка и компоненты менять не придётся.

## Фотографии товаров

Все загруженные вами фото медицинской формы уже разложены по карточкам мужской/женской линии в `lib/products.ts` и лежат в `public/products/`.

Фотографий поварской формы среди присланных изображений не было — сейчас на их месте временные фирменные заглушки (`chef-w-jacket.jpg`, `chef-m-jacket.jpg`) с подписью «Фото скоро появится». Просто добавьте свои реальные фото поварской формы в `public/products/` и обновите пути в `lib/products.ts` (или через админ-панель) — сайт сразу подхватит их.

## Деплой на GitHub + Vercel

1. Создайте пустой репозиторий на GitHub.
2. В папке проекта:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/<ваш-логин>/<название-репозитория>.git
   git push -u origin main
   ```
   `node_modules` и `.next` уже исключены через `.gitignore` — в репозиторий попадёт только исходный код.
3. Зайдите на [vercel.com](https://vercel.com), нажмите **Add New → Project**, выберите этот репозиторий — Vercel сам определит Next.js и всё соберёт. Отдельная настройка не нужна.
4. При каждом `git push` в `main` Vercel будет пересобирать и обновлять сайт автоматически.

Хостинг на своём сервере тоже возможен: после `npm run build` сайт запускается командой `npm start`. Статический экспорт не используется, так как каталог опирается на динамические маршруты `/product/[slug]`.

## Дальнейшее развитие

Код разделён на переиспользуемые компоненты и типы (`lib/types.ts`), поэтому легко добавить:

- корзину (несколько позиций в одном заказе);
- онлайн-оплату;
- личный кабинет клиента;
- реальный бэкенд для админ-панели.
